# Отладочный журнал

## Назначение

Журнал помогает проверить последовательность игровых событий и тайминги без просмотра исходного кода. Он выводит только важные события, а не данные каждого кадра.

## Как смотреть журнал

1. Запустите игру из редактора Godot.
2. После забега откройте нижнюю панель **«Вывод»**.
3. Найдите строки, начинающиеся с `[DEBUG]`.
4. Скопируйте нужные строки и отправьте их Codex вместе с описанием проблемы.

## Включение и выключение

- По умолчанию журнал **включён**.
- Во время запущенной игры нажмите `F3`, чтобы переключить его.
- В «Вывод» будет напечатано `[DEBUG][LOGGER] enabled` или `[DEBUG][LOGGER] disabled`.
- Чтобы изменить состояние по умолчанию, откройте `scripts/debug_log.gd` и поменяйте `var enabled := true` на `false`.

## Формат

```text
[DEBUG][ТЕГ] | детальная информация
```

## События текущего прототипа

| Тег | Когда появляется |
|---|---|
| `STATE` | Загружен или сохранён локальный рекорд. |
| `MENU` | Открыто меню, начат уровень, повтор или возврат в меню. |
| `RUN` | Начат, завершён или проигран забег. |
| `JUMP_INPUT` | Нажата клавиша прыжка; выводится позиция по X. |
| `LAND` | Герой приземлился; выводится позиция по X. |
| `FINISH_ZONE` | Герой пересёк зону финиша. |
| `RECORD` | Получен новый лучший результат. |
| `DAMAGE` | Кот получил урон от опасной зоны; выводятся оставшиеся сердца. |
| `COLLECT` | Подобрано письмо; выводится счёт писем. |
| `HEAL` | Подобрана рыбка; выводятся восстановленные и текущие сердца. |
