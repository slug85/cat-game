# Техническая архитектура

## Текущий вертикальный срез

```text
MainMenu
  └─ «Играть» → Level01
                    ├─ Player: автоматический бег и прыжок
                    ├─ крыши, препятствие и зона финиша
                    ├─ таймер забега
                    └─ победа / падение → повтор или меню

GameState (autoload)
  └─ локальное лучшее время в user://runner_save.cfg
```

## Сцены

- `scenes/main_menu.tscn` — точка входа приложения.
- `scenes/level_01.tscn` — первая фиксированная тестовая трасса.

Смена сцен выполняется из скриптов меню и уровня через `SceneTree`.

## Состояние

`GameState` подключён в `project.godot` как autoload. Он не зависит от конкретной сцены и хранит лучшее время между перезапусками приложения. Новые глобальные данные добавлять в этот скрипт только если они действительно нужны нескольким сценам.

## Границы ответственности

- `player.gd` отвечает только за движение и ввод.
- `level_01.gd` отвечает за правила конкретной трассы: таймер, падение, финиш и экран результата.
- `main_menu.gd` отвечает только за меню и навигацию.
- `game_state.gd` отвечает за локальное постоянное состояние.

## Проверка

После изменений сцен и скриптов запускать Godot в headless-режиме для проверки синтаксиса, затем открывать проект в редакторе для визуальной проверки трассы.
